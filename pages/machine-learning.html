Credit card fraud detection is one of the most challenging real-world ML problems — not because of model complexity, but because of extreme class imbalance. Fraudulent transactions are rare, so a model that simply predicts "legitimate" for everything would achieve very high accuracy while being completely useless.

The dataset — real European credit card transactions from Kaggle — reflects this: the vast majority are legitimate, with fraud cases representing a tiny fraction. This required using precision, recall, F1-score, and ROC-AUC as primary metrics rather than raw accuracy.

Logistic Regression was selected as the primary algorithm for its efficiency, interpretability, and excellent performance on binary classification tasks. It provides probabilistic outputs — meaning the system doesn't just say "fraud" or "not fraud", it gives a confidence score, which is far more useful for risk assessment.

The Gradio deployment transforms the model from a notebook into a usable application — users input transaction details and receive an instant prediction with fraud probability. This makes the system educational and demonstrates real-world deployment thinking.

// Reflection

Key Learnings

01

ML is iterative — every result reveals what you don't yet understand

Each model trained in the experiments opened a new question. Linear Regression exposing non-linearity. Random Forest outperforming without tuning. Hyperparameter changes shifting behaviour significantly. The learning isn't in the final model — it's in the progression.

02

Accuracy is a misleading metric for imbalanced data

A fraud detection model that predicts "legitimate" for everything would achieve 99%+ accuracy while being completely useless. Precision, recall, F1-score, and ROC-AUC are the real measures of performance when classes are imbalanced — and this was the most important conceptual shift in A2.

03

Feature importance reveals what the model actually learned

In the gaming classification task, gameplay metrics (PlayTimeHours, AvgSessionDuration) were far more predictive than demographic features. Feature importance analysis turned the model from a black box into an explainable system — and that explanation was itself valuable insight.

04

Deployment is what separates a notebook from a product

Building the Gradio interface for the fraud detection model transformed it from a Jupyter notebook into something a non-technical person could use. Deployment forces you to think about the user, not just the model — and that thinking makes the model better.

05

Preprocessing often matters more than model choice

In both assessments, the most impactful decisions were in data preparation — encoding strategy, scaling method, stratified splitting, handling class imbalance. A well-preprocessed dataset with a simple model consistently outperformed a complex model on poorly prepared data.
