Concept & Goal

ColorMix started with a simple question: how many people see colours every day but don't understand how they mix? The app turns that curiosity into an interactive experience — tap two primary colours and see the result instantly, without reading a word of theory.

Designed for school students, hobbyists, and visual learners of all ages — lightweight, fast, and accessible. The cleaner the interface, the better the experience, especially for learning through interaction.

Design Approach

Three main sections on screen at all times: colour selection buttons, a mix display area, and a reset option. Bright high-contrast colours, lots of white space, large tap-friendly buttons. The app looks playful but stays professional — every element placed so the user can explore without needing instructions.

// Technical

Key Decisions

Decision 01

RGB Averaging Blending Logic

Primary colours are converted to RGB values and averaged to generate the mixed result. Not as scientifically accurate as pigment-based mixing, but works perfectly for screen-based visual feedback — and instant enough to feel responsive.

Decision 02

Single-Page Jetpack Compose Layout

Built as a single-page reactive UI — allowing for quicker load times and better state management. Compose's declarative approach meant fewer bugs when updating colour state in real time, with minimal boilerplate.

Decision 03

Tap-to-Mix — No Sliders

Deliberately avoided sliders or complex input tools. A tap interaction is more intuitive, less overwhelming, and faster — matching the educational goal of showing results immediately without friction. Subtle animations keep it responsive without overloading performance.

Concept & Goal

FitTrack is a virtual accountability partner — not a complex planner, but a lightweight, clean tool that makes it easy to log workouts and build the habit of tracking progress. Whether you're a beginner or an experienced gym-goer, the goal is to make fitness feel less overwhelming.

Three screens: a home screen showing daily activity, a log screen for adding new workouts, and a history screen for reviewing past performance. Every interaction is purposeful — nothing extra.

Key Decisions

Decision 01

Offline-First with Room Database

Many users won't always have internet access — so local storage was the priority from day one. Room Database integrates cleanly with Compose and ensures workout data is always available, regardless of network status. No login, no cloud dependency.

Decision 02

State Hoisting for UI Consistency

Updating workout logs and switching between screens can cause UI inconsistencies if state isn't managed carefully. State hoisting techniques were used throughout to manage UI consistency and avoid unnecessary recompositions — keeping the app responsive and predictable.

Decision 03

Scalable Architecture — No Login Yet

Deliberately avoided a login system or cloud syncing at this stage, keeping the focus on usability. However, the app's architecture is built to scale — Firebase integration, reminders, and calorie tracking can all be added without rebuilding from scratch.

// Reflection

Key Learnings

01

Jetpack Compose changes how you think about UI

Declarative UI removes the mental overhead of manually syncing view state. Once the pattern clicks — describe what the UI should look like given a state, not how to update it — building reactive interfaces becomes dramatically faster and less error-prone.

02

Simplicity is a design decision, not a limitation

Both apps deliberately left features out. ColorMix skipped sliders; FitTrack skipped login. These weren't gaps — they were choices that kept the user experience clean and the codebase focused. Knowing what to remove is as important as knowing what to build.

03

Offline-first is the right default for mobile

Designing around Room Database from the start — instead of retrofitting local storage later — made FitTrack inherently more reliable. Building mobile apps that assume connectivity is a mistake; building for offline-first produces apps people actually trust.

04

State management is the hardest part of any app

State hoisting in FitTrack — and real-time state updates in ColorMix — were the most technically demanding aspects. Getting state management right is what separates a working prototype from a production-quality app.

05

Build for scale from day one

FitTrack's architecture was designed so that Firebase, reminders, and calorie tracking could be added without rebuilding. Writing scalable code when you're building something small isn't overengineering — it's the difference between a project and a product.
