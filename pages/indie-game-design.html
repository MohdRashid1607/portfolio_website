// Overview

Flappy Bird — Built from Scratch

4
Live on itch.io — playable in browser Unity · C# · Rigidbody2D Spacebar to fly

Flappy Bird is a deceptively simple game — the entire mechanic is one button. But building it from the ground up in Unity requires every system to be engineered precisely: physics, collision, pipe spawning, scoring, and restart logic all need to work together without breaking.

The player controls a bird navigating through the gaps between pairs of pipes by pressing the spacebar. Each successful pass scores one point. Touch a pipe, hit the ground, or stop pressing — and the bird falls and dies. The game resets from zero with no time limit and no upper bound on score.

"Adjusting the game's difficulty was one of the main challenges. At first the pipe spacing was irregular — through iteration and testing, I found the balance that makes it feel both fair and challenging."

— Critical Reflection, M.Rashid

// Technical

5 C# Scripts

Core

GameManager.cs

The main control mechanism for the entire game. Uses the Singleton pattern to ensure one instance only. Manages all key game states — start, pause, end — and handles score display, the game-over panel, and restart logic by resetting the time scale and repositioning all elements on death.

Player

Player.cs

Handles the bird's movement, sprite animation, and collision interactions. Cycles through an array of sprites at set intervals to simulate flapping motion. Applies upward force via Rigidbody2D on spacebar press — with gravity pulling the bird down continuously when no input is given.

Obstacles

Pipes.cs

Controls the movement and lifecycle of pipe obstacles. Pipes scroll left at constant speed and are destroyed on exit — preserving performance by preventing object buildup in the scene.

Spawning

Spawner.cs

Dynamically instantiates pipe prefabs at regular intervals with randomised heights — ensuring consistent challenge. Spawn rate is adjustable for difficulty tuning without touching core logic.

Visual

Parallax.cs

Creates the scrolling background effect — different layers scroll at different speeds, simulating depth and reinforcing the illusion of flying through a real environment.

// Reflection

Key Learnings

01

Simple mechanics hide complex systems

Flappy Bird looks like a one-button game. Building it required 5 interconnected scripts, physics tuning, collision layers, and spawn logic. Real simplicity on screen requires serious engineering underneath.

02

Player feedback is the best debugging tool

The pipe clipping bug and the difficulty imbalance both came from playtester observation — not code review. Watching 4 real players revealed things no test plan alone would have caught.

03

Rigidbody2D physics demands careful tuning

Getting the bird's gravity, upward force, and idle falling to feel right took more iteration than any other system. Physics-based feel is subjective — and that subjectivity is what makes it hard to code correctly.

04

Game feel is a design discipline, not polish

The parallax background, sprite animation, and sound effects weren't cosmetic — they were what turned a functional prototype into an enjoyable game. Responsiveness and feel matter as much as the core mechanics.

05

Publishing makes the project real

Getting the game live on itch.io wasn't just a submission step — it was the point. Shipping to a real platform, writing a game page, and collecting live feedback was the final stage that completed the project.
