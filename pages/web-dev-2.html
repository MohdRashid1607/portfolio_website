// Process

5 Key Development Decisions

1

Responsive Header — horizontal on desktop, collapsed on mobile

The header includes the logo and navigation bar displayed horizontally on desktop screens. For tablet and mobile viewports, the navigation collapses into a hamburger-style menu — a deliberate decision to prioritise screen real estate on smaller devices without sacrificing discoverability.

2

Two-column intro layout that stacks on mobile

The introduction section uses a two-column grid on larger screens — image on one side, description on the other. On smaller devices, this collapses to a single stacked column. This was chosen over a single-column layout across all sizes to make better use of the wider screen real estate on desktops.

3

Authentication forms moved to a separate page

Initially, the sign-up and login forms were integrated into the homepage. After testing, they were moved to a dedicated authentication page — reducing visual clutter on the home screen and giving the forms more focused design attention. Passwords are hashed with bcrypt before storage in MongoDB.

4

Media queries throughout — grids on desktop, stacked on mobile

Media queries were applied extensively to adjust layouts, font sizes, and spacing at each of the four breakpoints. Grid layouts used on desktop and tablet screens were replaced with vertical stacks on mobile — ensuring each device feels purpose-built rather than merely scaled down.

5

MongoDB Atlas chosen for scalable cloud data storage

MongoDB Atlas was selected over a local database for its scalability, ease of cloud integration, and native compatibility with Mongoose schemas. Peer-based schema validation was implemented to enforce data integrity on user registration and login endpoints.

// Reflection

Critical Analysis

✓ Successes

Responsive & Accessible

The website adapts seamlessly to all four device sizes — responsive layout, consistent theming, and the dark mode toggle all received positive feedback. The core bookshop functionality works reliably across all breakpoints.

⚠ Limitations

Missing Engagement Features

The absence of book reviews, user ratings, or personalised recommendations limits user engagement. The payment and shopping cart flow is incomplete — users can browse but can't purchase. Search and filtering could be deeper.

→ Future Improvements

Cart, Payments & Reviews

A shopping cart with payment integration (Stripe), a user review system, and personalised recommendations would significantly increase the site's competitiveness. Improved search and filter functionality would also enhance the browsing experience.

// Takeaways

Key Learnings

01

Cloning teaches more than building from scratch

Reproducing LinkedIn's homepage forced a deep reading of every layout decision — spacing, colour, typography hierarchy, nav behaviour. You can't fake pixel-perfect reproduction. Studying then rebuilding is one of the most efficient ways to level up CSS skills.

02

Full-stack means thinking in layers

The bookshop app required frontend, backend, database, authentication, and deployment to work together. A bug in one layer surfaces as a symptom in another. Understanding how each layer affects the others is what separates a full-stack developer from a frontend one.

03

Responsive design is architecture, not an afterthought

Supporting four breakpoints from day one shaped every layout decision — two-column vs stacked, hamburger vs full nav, grid vs flex. Responsive design that works at every size has to be planned before the first div is written, not added at the end.

04

Security belongs in the foundation

Implementing bcrypt password hashing, environment variables for credentials, and CORS configuration wasn't optional — it was part of the brief. These security decisions can't be retrofitted later without significant rework.

05

Deployment is part of development

Getting the app live on Vercel with a working MongoDB Atlas connection required understanding environment variables, build pipelines, and production vs development config. Shipping to a real URL was when the project truly felt finished.
